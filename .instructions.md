# Always Auto Commit, Push, and Sync

## Rule
- After every code change, automatically commit the change with an appropriate message.
- Immediately push the commit to the remote repository.
- Always keep the local and remote repositories in sync.
- Do not wait for manual user commands to commit or push.

## Rationale
This ensures that all progress is saved and synchronized, reducing the risk of data loss and keeping the project up to date across all environments.

## Example Prompts
- "Add a new feature to the game."
- "Fix a bug in the 3D rendering."
- "Update the UI style."

## Related Customizations
- Enforce commit message conventions
- Require branch sync before edits
- Auto-resolve simple merge conflicts
